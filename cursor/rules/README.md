# Generic Cursor rules (fragments)

Portable sections for DBQ-style **documentation and planning** workflows. Copy one file into a repo’s `.cursorrules`, or merge multiple fragments. Replace every `{{PLACEHOLDER}}` with values for that project.

| File | Use when |
|------|----------|
| [`implementation-plans.md`](implementation-plans.md) | Issue-linked markdown plans next to code (or your documented location) |
| [`architecture-versioning.md`](architecture-versioning.md) | Architecture docs split by product version |
| [`references-in-plans.md`](references-in-plans.md) | How plans should cite code, architecture, and requirements |
| [`best-practices.md`](best-practices.md) | Writing actionable plans |
| [`planning-checklists.md`](planning-checklists.md) | Multi-tenancy, security, and performance prompts for plan authors |

For a **single file** that combines the above with placeholders, use [`../templates/cursorrules.planning-repository.md`](../templates/cursorrules.planning-repository.md).

The minimal empty shell remains [`../templates/cursorrules.template.md`](../templates/cursorrules.template.md).
